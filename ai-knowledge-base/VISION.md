# AI 知识库系统 — 项目愿景文档

## 1. 项目使命

**让 AI 领域的每日进展，自动变成你的结构化知识资产。**

每天有数百篇论文、数千个开源项目和无数技术讨论诞生。一个 AI 从业者即使全天泡在 arXiv、Hacker News 和 GitHub 上，也只能覆盖冰山一角。更关键的是，**看到 ≠ 学到** — 碎片信息看过就忘，无法沉淀为自己的知识。

这个系统要做的事情很明确：**用 Agent 替代人工完成"采集→筛选→理解→整理→发布"全流程**，让你每天花 10 分钟就能掌握 AI 领域真正重要的进展。

## 2. 目标用户

| 用户类型 | 核心痛点 | 使用场景 |
|---------|---------|---------|
| AI 工程师 | 信息过载，没时间每天跟踪最新论文和开源项目 | 每天早上看日报，了解值得关注的新技术和论文 |
| 技术管理者 | 需要把握技术趋势但无法深入细节 | 每周看周报，获取趋势概览和关键技术判断 |
| AI 研究者 | 需要追踪特定子领域的最新进展 | 按主题订阅，获取领域内的最新论文和复现代码 |

## 3. 核心价值主张

**一句话：** 把 AI 领域的信息洪流，变成每天一份可读、可信、可追溯的结构化知识日报。

三个关键词：

- **自动化** — 无需人工干预，定时抓取、自动分析、自动发布
- **结构化** — 不是链接合集，而是带摘要、分类、关联的结构化知识
- **可信任** — 每个条目经过 Agent 多轮校验，标注来源和置信度

## 4. 系统全景

```
数据源                  采集层              分析层              整理层              发布层
┌──────────┐      ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ GitHub   │─────▶│              │    │              │    │              │    │              │
│ Trending │      │  采集 Agent   │───▶│  分析 Agent   │───▶│  整理 Agent   │───▶│  发布 Agent   │───▶ 日报/周报
├──────────┤      │              │    │              │    │              │    │              │
│ Hacker   │─────▶│ · 抓取       │    │ · 分类       │    │ · 去重合并   │    │ · Markdown   │
│ News     │      │ · 去重       │    │ · 摘要       │    │ · 关联发现   │    │ · 静态站点   │
├──────────┤      │ · 筛选       │    │ · 质量评分   │    │ · 知识图谱   │    │ · RSS/订阅    │
│ arXiv    │─────▶│              │    │              │    │              │    │              │
└──────────┘      └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                         │                   │                   │                   │
                    原始数据层           分析数据层           知识数据层           发布产物
                    raw/                analyzed/            organized/           output/
```

流水线四个阶段各由一个独立 Agent 负责，Agent 之间通过文件系统传递数据，松耦合、可独立调试和替换。

## 5. 关键能力（Phase 1）

### 5.1 智能采集
- 定时抓取三大数据源的热门 AI 相关内容
- 自动去重（同一篇论文可能同时在 HN 和 arXiv 出现）
- 关键词过滤，只保留 AI/ML 相关条目

### 5.2 自动分析
- 对每条内容生成中文摘要（100-200 字）
- 自动归类（大语言模型 / 计算机视觉 / 强化学习 / 工具框架 / 行业动态 …）
- 质量评分（1-5 星），基于社区反响和内容深度

### 5.3 知识整理
- 合并相关条目（例如：论文 + 对应的开源实现 + HN 讨论）
- 发现跨条目关联（"本周有 3 个项目都在做 Agent 框架"）
- 剔除低质量或与 AI 无关的内容

### 5.4 多渠道发布
- 每日 AI 日报（Markdown，可推送到 Obsidian / Notion / 公众号）
- 每周趋势周报（Top 10 盘点 + 趋势总结）
- 支持按主题/来源/时间检索历史

## 6. 设计原则

1. **Agent 原生** — 每个环节都设计为可由 Agent 独立完成，人工只做最终审核
2. **数据不可变** — 原始抓取数据永久保留，分析和整理结果可随时回炉重算
3. **渐进增强** — 先跑通"每日链接合集"的最小闭环，再逐步加摘要、分类、趋势分析
4. **可观测** — 每个 Agent 的输入输出可追溯，质量下降时能快速定位问题环节

## 7. 成功指标

| 指标 | 目标（Phase 1） | 衡量方式 |
|------|----------------|---------|
| 全流程自动化率 | 100% | 无需人工介入即可产出日报 |
| 内容相关性 | ≥ 90% | 人工抽查 100 条，标记无关条目占比 |
| 摘要可用性 | ≥ 80% | 用户反馈"摘要准确、无需点原文"的占比 |
| 日报阅读时间 | ≤ 10 分钟 | 用户读完日报的平均时间 |
| 覆盖完整度 | 无重大遗漏 | 与手工跟踪的对照组对比召回率 |

## 8. 分阶段路线图

### Phase 1 — 最小闭环（MVP）
- 单一数据源（GitHub Trending）
- 采集 → 分析 → 发布三段式流水线
- 输出：每日 AI 开源项目速览（Markdown 文件）
- **目标：跑通全流程，验证 Agent 流水线可行性**

### Phase 2 — 多源融合
- 接入 Hacker News 和 arXiv
- 增加去重和跨源关联
- 引入质量评分机制
- **目标：日更一份综合性 AI 日报**

### Phase 3 — 智能增强
- 主题订阅（用户可指定关注领域）
- 趋势发现（"本周最值得关注的 3 个信号"）
- 历史回溯（支持检索已入库的所有条目）
- **目标：从"日报"升级为"可查询的知识库"**

### Phase 4 — 社区化
- 静态站点自动发布
- RSS / 邮件订阅
- 开放 PR（社区可提交新数据源或分析规则）
- **目标：服务更多人，接受社区反馈迭代**
